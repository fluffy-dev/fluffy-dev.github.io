система автоматизации e-commerce (afterbuy)

высокопроизводительная экосистема для парсинга, обработки и управления данными, которая сократила время ручных операций с 1 часа до 5 минут и обрабатывает >100,000 запросов в день.

кликни на картинку, чтобы увеличить.

техническое погружение:

архитектурные решения
  • слойная архитектура: строгое разделение на `routers`, `services` и `repositories` для изоляции логики и упрощения тестирования.
  • dependency injection: активное использование `fastapi.depends` для управления зависимостями и сессиями бд, что обеспечивает чистоту кода.
  • lazy-инициализация бд: кастомный `databasehelper` для `sqlalchemy`, который создает `engine` только при первом вызове, что критически важно для совместимости с `celery`.
  • reverse-engineering: вместо хрупкого парсинга html был использован анализ http-запросов и внутреннее api afterbuy.
масштабируемость и производительность
  • асинхронный пайплайн: `fastapi` + `aiohttp` + `asyncpg` обеспечивают полностью неблокирующий i/o, позволяя обрабатывать сотни одновременных запросов.
  • фоновая обработка: все долгие задачи (>1 сек) вынесены в `celery`, что гарантирует мгновенный ответ api.
  • агрессивное кэширование: `redis` используется для кэширования ответов api, данных о продуктах и сессий, что снижает нагрузку на бд в десятки раз.
  • rate limiting: `slowapi` защищает сервис от ddos-атак и чрезмерной нагрузки от клиентов.
безопасность
  • jwt-аутентификация: с refresh-токенами и разделением ролей (user/admin).
  • хэширование паролей: используется `argon2`, один из самых криптостойких алгоритмов.
  • валидация данных: `pydantic` используется на всех уровнях для строгой валидации входящих и исходящих данных, предотвращая инъекции и ошибки.
  • cors: тонкая настройка cors-политик для безопасного взаимодействия с фронтендом.
devops и observability
  • контейнеризация: все компоненты системы (api, celery, postgres, redis) обернуты в `docker` и управляются через `docker-compose`.
  • мониторинг: полный стек наблюдаемости (`prometheus`, `grafana`, `loki`, `tempo`) через `opentelemetry`.
  • ci/cd (план): настроить github actions для автоматического прогона тестов (`pytest`), линтинга (`black`, `ruff`) и сборки docker-образов.

the loom: real-time collaborative ide

pet-проект для исследования conflict-free replicated data types (crdt). это браузерная ide, где несколько пользователей могут редактировать код одновременно, как в google docs.

кликни на картинку, чтобы увеличить.

техническое погружение:

алгоритмическое ядро
  • реализация crdt: реализованы алгоритмы `logoot` и `lseq` для текстовых данных. `logoot` был выбран за баланс между метаданными и производительностью.
  • фракционные идентификаторы: каждый символ имеет уникальный, плотно упакованный позиционный идентификатор, что позволяет вставлять символы между любыми двумя без перенумерации.
  • tombstones: для обработки удалений используются "надгробия" (tombstones), чтобы сохранить историю и корректно применять операции, пришедшие с опозданием.
низкоуровневая оптимизация
  • binary protocol: для минимизации задержек и размера полезной нагрузки используется бинарный протокол (`protocol buffers`) поверх `websockets`, вместо json. это сократило трафик на ~40%.
  • batching операций: клиент не отправляет каждую операцию (нажатие клавиши) немедленно, а собирает их в пачки и отправляет каждые 100 мс или при неактивности, снижая сетевую нагрузку.
  • zero-copy parsing: на сервере, где это возможно, используются техники zero-copy для парсинга бинарных сообщений, минимизируя копирование данных в памяти.
архитектура и отказоустойчивость
  • state vectors: реализован механизм векторов состояний для быстрой синхронизации при подключении нового клиента или после потери соединения, передавая только дельту изменений.
  • изоляция сессий: каждая сессия редактирования изолирована в отдельном процессе `celery` с собственным каналом в `redis pub/sub`.
  • снапшоты в redis: сервер периодически делает снапшоты документа в `redis`, что позволяет мгновенно восстанавливать состояние сессии при сбое воркера.

graph visualization & algorithms

интерактивный инструмент для визуализации графов и пошагового выполнения классических графовых алгоритмов, написанный с использованием низкоуровневых библиотек рендеринга.

кликни на картинку, чтобы увеличить.

техническое погружение:

gpu-ускоренный рендеринг
  • opengl pipeline: для отрисовки тысяч вершин и ребер используется современный `opengl` с кастомными шейдерами (glsl).
  • vbo & ebo: данные о геометрии графа (вершины, цвета, ребра) загружаются напрямую в память видеокарты через `vertex buffer object` и `element buffer object`, минимизируя обмен с cpu.
  • instanced rendering: для отрисовки одинаковых элементов (например, маркеров вершин) используется техника `instanced rendering`, позволяющая нарисовать тысячи объектов за один вызов api.
эффективные алгоритмы и структуры данных
  • fibonacci heap: для алгоритма дейкстры используется `fibonacci heap` вместо бинарной кучи для асимптотического ускорения на плотных графах (o(e + v log v) -> o(e + v log v)).
  • disjoint set union (dsu): для алгоритма краскала (построение минимального остовного дерева) реализована структура dsu с оптимизациями "path compression" и "union by rank".
  • пространственное индексирование: для быстрого поиска вершины по клику мыши (picking) используется `quadtree`.
