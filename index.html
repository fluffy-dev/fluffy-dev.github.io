Ключевые архитектурные решения:

Слойная архитектура (Layered Architecture)

Приложение четко разделено на слои: **Routers** (прием запросов), **Services** (бизнес-логика) и **Repositories** (доступ к данным). Это упрощает тестирование, поддержку и независимую замену компонентов.

Асинхронность и фоновая обработка

Для долгих задач (парсинг) используется **Celery** и **RabbitMQ**. API мгновенно отвечает клиенту, а вся тяжелая работа выполняется в фоне, что исключает таймауты и обеспечивает отзывчивость системы.

Reverse-Engineering вместо парсинга HTML

Вместо хрупкого парсинга верстки был проведен анализ и использовано внутреннее API платформы Afterbuy. Это обеспечило на порядок большую скорость и стабильность получения данных.

Полная наблюдаемость (Observability)

С помощью стека **OpenTelemetry, Prometheus, Grafana, Loki и Tempo** система предоставляет исчерпывающую информацию о своем состоянии: метрики производительности, распределенные трейсы запросов и структурированные логи.
